4. Ответы на вопросы
	1. Объект ядра - это структура данных в памяти ядра Windows, которая представляет системные ресурсы и обеспечивает унифицированный доступ к ним через Object Manager.
	Некоторые типы объектов ядра в Windows:
	- маркеры доступа (access token objects);
	- файлы (file objects);
	- проекции файлов (file-mapping objects);
	- порты завершения ввода-вывода (I/O completion port objects);
	- задания (jobs);
	- почтовые ящики (mailslot objects);
	- мьютексы (mutex objects);
	- каналы (pipe objects);
	- процессы (thread objects);
	- семафоры (semaphore objects);
	- потоки (thread objects);
	- ожидаемые таймеры (waitable timer objects).
	2. **Утечка HANDLE** происходит, когда приложение получает дескриптор (handle) к ресурсу, но не закрывает его после использования.
	3. **Чем отличается закрытие окна приложения от завершения процесса в плане HANDLE и объектов ядра?**
	Закрытие окна (WM_CLOSE):
	- Отправляет сообщение окну для корректного завершения.
	- Приложение может обработать сообщение, сохранить данные и освободить ресурсы.
	- Handles закрываются корректно самим приложением.
	- Объекты ядра уничтожаются правильно.
	Завершение процесса (TerminateProcess):
	- Handles к объектам ядра закрываются автоматически при завершении процесса.
	- Но сами объекты ядра не уничтожаются немедленно - они существуют до тех пор, пока все процессы, имеющие открытые handles к ним, не закроют эти handles.
	- Не дает приложению возможности сохранить данные или корректно освободить ресурсы
	- Может оставить ресурсы в непредсказуемом состоянии.

## Чем **командлеты** принципиально отличаются от обычных команд?
**Командлеты** — это специализированные команды, написанные на .NET, которые выполняют определённое действие, используя входные данные и выдавая результаты в виде объектов.
